### UI Components

1. **Header Section**
   - Page title: "Expense Policies"
   - Primary action button: "+ Create Policy"

2. **Filter Bar**
   - Search input (searches name and code)
   - Status dropdown: All, Active, Inactive

3. **Policy Cards**
   - Policy icon and name
   - Policy code
   - Badges: [DEFAULT] for default policy, [ACTIVE]/[INACTIVE] for status
   - Quick info: Categories count, Monthly limit
   - Applicability summary
   - Receipt and past days info
   - Action buttons: View, Edit, Clone, More (...)

4. **More Menu (...)** Options:
   - Set as Default Policy
   - Duplicate Policy
   - Activate / Deactivate
   - Delete (if not in use)

5. **Quick Actions Bar**
   - Find Employee Policy (test which policy applies to an employee)
   - Policy Priority Order (view/edit priority)

## Form Field Validations

### Basic Information
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| policy_name | text | Yes | Max 200 chars |
| policy_code | text | Yes | Max 50 chars, unique, auto-uppercase |
| policy_description | textarea | No | Max 2000 chars |
| priority | number | No | 0-100, default 0 |
| is_default | checkbox | No | Only one default allowed |
| is_active | checkbox | No | Default true |

### Spending Limits
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| overall_limit_per_transaction | currency | No | Min 0, max 15 digits |
| overall_limit_per_day | currency | No | Min 0, must be >= per_transaction |
| overall_limit_per_week | currency | No | Min 0, must be >= per_day |
| overall_limit_per_month | currency | No | Min 0, must be >= per_week |
| overall_limit_per_quarter | currency | No | Min 0, must be >= per_month |
| overall_limit_per_year | currency | No | Min 0, must be >= per_quarter |

### Submission Controls
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| max_past_days | number | No | 1-365, default 30 |
| max_future_days | number | No | 0-30, default 0 |
| submission_window_start_day | number | No | 1-31 |
| submission_window_end_day | number | No | 1-31, must be > start_day |
| require_receipt_above | currency | No | Min 0 |

### Duplicate Detection
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| duplicate_check_days | number | No | 1-90, default 7 |
| duplicate_check_fields | multiselect | No | At least 2 fields if enabled |

### Applicability Rules
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| applicability_type | dropdown | Yes | Must be valid type |
| applicability_value | multiselect | Conditional | Required for most types |
| priority | number | No | 0-100, default 0 |

## Error States & Messages

### Create/Update Errors
```javascript
// Duplicate code error
{
    "success": false,
    "message": "A policy with this code already exists"
}

// Default policy constraint
{
    "success": false,
    "message": "Cannot set as default - another active default policy exists. Please deactivate it first."
}

// Validation error
{
    "success": false,
    "message": "Policy name is required"
}
```

### Delete Errors
```javascript
// Default policy
{
    "success": false,
    "message": "Cannot delete the default policy. Please set another policy as default first."
}

// Policy in use (future implementation)
{
    "success": false,
    "message": "Cannot delete policy - it has active expense requests"
}
```

## Notes for Frontend Developer

1. **Priority Logic**: Higher priority policies are checked first. Employee-specific rules (priority 10) override department rules (priority 5).

2. **Category Selection**: Use a tree-view component for category selection since categories have parent-child relationships.

3. **Spending Limits Validation**: Ensure hierarchical validation (yearly >= quarterly >= monthly >= weekly >= daily >= per_transaction).

4. **Applicability Rules**: Multiple rules work with OR logic (employee matches ANY rule). Exclusion rules are checked first.

5. **Default Policy**: Only one default policy can exist. Handle the toggle appropriately.

6. **Form State**: Use a stepper/wizard pattern for the form. Allow saving drafts at any step.

7. **Currency Amounts**: Format with locale-specific separators (e.g., â‚¹1,00,000 for INR).

8. **Testing Tool**: The "Find Employee Policy" feature is crucial for admins to verify policy configurations.
