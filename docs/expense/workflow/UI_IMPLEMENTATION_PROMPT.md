### UI Components

1. **Header Section**
   - Page title: "Expense Approval Workflows"
   - Primary action button: "+ Create Workflow"

2. **Filter Bar**
   - Search input (searches name and code)
   - Scope dropdown: All, All_Expenses, Category_Specific, Amount_Based, Policy_Specific
   - Status dropdown: All, Active, Inactive

3. **Workflow Cards**
   - Workflow icon and name
   - Workflow code
   - Badges: [DEFAULT] for default workflow, [ACTIVE]/[INACTIVE] for status
   - Quick info: Scope, Mode, Approval Level
   - Visual stage pipeline showing stages
   - Statistics: Active requests count, Average completion time
   - Action buttons: View, Edit, Clone, More (...)

4. **More Menu (...)** Options:
   - Set as Default Workflow
   - View Analytics
   - Duplicate Workflow
   - Export Configuration
   - Activate / Deactivate
   - Delete (if no active requests)

5. **Quick Actions Bar**
   - Category Mappings (opens mapping screen)
   - View Analytics (opens dashboard)

## Form Validation Rules

### Workflow Form

| Field | Validation |
|-------|------------|
| workflow_name | Required, max 150 chars |
| workflow_code | Required, max 50 chars, unique per company, uppercase alphanumeric + underscores |
| workflow_scope | Required, must be valid enum |
| approval_mode | Required, must be valid enum |
| approval_level | Required, must be valid enum |
| max_amount_reduction_percentage | 0-100 |
| escalation_after_hours | Positive integer |
| max_escalation_levels | 1-10 |
| max_send_back_count | 1-10 |
| stages | At least 1 stage required |

### Stage Form

| Field | Validation |
|-------|------------|
| stage_order | Required, positive integer |
| stage_name | Required, max 100 chars |
| approver_type | Required, must be valid enum |
| approver_user_ids | Required when approver_type = Specific_User |
| approver_role_id | Required when approver_type = Specific_Role |
| min_amount | >= 0 |
| max_amount | > min_amount (if specified) |
| sla_hours | Positive integer |
| sla_warning_hours | < sla_hours |
