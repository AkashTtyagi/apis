### UI Components

1. **Header Section**
   - Page title: "Currency Management"
   - Primary action button: "+ Add Currency"

2. **Filter Bar**
   - Search input (searches code and name)
   - Status dropdown: All, Active, Inactive

3. **Currency Cards**
   - Large currency symbol
   - Currency code and full name
   - Badges: [BASE] for base currency, [DEFAULT] for default expense currency
   - Display settings (decimal places, symbol position)
   - Current rate to base currency (if not base)
   - Status indicator (green dot = active, gray = inactive)
   - Action buttons: Edit, Rates, More (...)

4. **More Menu (...)** Options:
   - Set as Base Currency
   - Set as Default Currency
   - View Rate History
   - Deactivate / Activate
   - Delete (if no expenses)

5. **Quick Actions Bar**
   - Set Base Currency (opens selection modal)
   - Currency Policy (opens policy settings)
   - Bulk Rates (opens bulk rate update)

6. **Pagination**
   - Standard pagination controls

### API Integration

**On Page Load:**
```
POST /api/expense/admin/currencies/list
Body: {
    "is_active": null,
    "search": "",
    "limit": 50,
    "offset": 0,
    "sort_by": "currency_code",
    "sort_order": "asc"

```

**On Search/Filter Change:**
- Debounce search input (300ms)
- Call list API with updated filters

**On Delete Click:**
```
POST /api/expense/admin/currencies/delete
Body: { "currency_id": <id> }
```

**On Set Base Currency:**
```
POST /api/expense/admin/currencies/set-base
```

### Form Fields

| Field | Type | Required | Validation |
|-------|------|----------|------------|
| currency_code | Text (uppercase) | Yes | 3 letters, ISO 4217 format, unique per company |
| currency_name | Text | Yes | Max 100 characters |
| currency_symbol | Text | Yes | Max 10 characters |
| currency_symbol_position | Dropdown | Yes | Before, After |
| country_id | Dropdown | No | From countries list |
| decimal_places | Dropdown | Yes | 0, 1, 2, 3, 4 |
| decimal_separator | Dropdown | No | Period (.), Comma (,) |
| thousands_separator | Dropdown | No | Comma (,), Period (.), Space, None |
| is_base_currency | Checkbox | No | Boolean |
| is_default_expense_currency | Checkbox | No | Boolean |
| is_active | Radio | Yes | Active/Inactive |

### Form Behavior

1. **Currency Code Field**
   - Auto-uppercase input
   - Validate 3-letter format
   - Check uniqueness on blur (call API or client-side check)
   - Disabled in edit mode

2. **Symbol Position Preview**
   - Show preview based on selected position
   - "Before": $100.00
   - "After": 100.00$

3. **Live Format Preview**
   - Update preview when decimal_places, decimal_separator, or thousands_separator changes
   - Show sample amount: 1234567.89 formatted

4. **Base Currency Warning**
   - Show warning when checking "Set as Base Currency"
   - Display current base currency name

5. **Edit Mode Restrictions**
   - Currency code is readonly
   - Show warning if currency is in use

### API Integration

**On Form Open (Edit Mode):**
```
POST /api/expense/admin/currencies/details
Body: { "currency_id": <id> }
```

**On Form Open (Get Dropdown Data):**
```
POST /api/expense/admin/currencies/dropdown
Body: { "include_inactive": false }
```

```
Body: {
```

   - Show warning with existing rate details

### Form Behavior

1. **Conditional Fields**
   - If "Allow Multi-Currency" is OFF, disable conversion settings
   - If "Allow Manual Rate Override" is OFF, disable "Require Justification"

2. **Live Preview**
   - Rounding preview updates with method/precision change
   - Display preview updates with toggle changes

3. **Conversion Timing Explanation**
   - At Submission: Rate locked when expense is submitted
   - At Approval: Rate applied when manager approves
   - At Payment: Rate applied at reimbursement time

## Validation Rules

### Currency Validation
- Currency code: 3 uppercase letters (ISO 4217)
- Currency code: Unique per company
- Currency symbol: Max 10 characters
- Decimal places: 0-4
- Cannot delete base currency
- Cannot delete currency with expenses

### Exchange Rate Validation
- Rate must be > 0
- From and To currencies must be different
- Both currencies must be active
- Effective_from is required
- No overlapping periods for same currency pair

### Policy Validation
- Rate tolerance: 0-100%
- Max rate age: 1-365 days
- Rounding precision: 0-4

## Error Messages

| Error Code | Message | Action |
|------------|---------|--------|
| CURR_001 | Currency code already exists | Use different code |
| CURR_002 | Invalid currency code format | Use 3-letter ISO code |
| CURR_003 | Cannot delete base currency | Set different base first |
| CURR_004 | Currency has existing expenses | Cannot delete |
| RATE_001 | Exchange rate must be positive | Enter valid rate |
| RATE_002 | No rate found for date | Add rate for date |
| RATE_003 | Rate expired | Add new rate |
| RATE_004 | Currencies must be different | Select different pair |
| POL_001 | Invalid tolerance percentage | Enter 0-100 |

## User Flows

### Flow 1: Add New Currency
1. Click "+ Add Currency" button
2. Fill currency information
3. Set display options
4. Optionally set as base/default
5. Click Save
6. Success: List refreshes with new currency
7. Failure: Show validation errors

### Flow 2: Update Exchange Rate
1. Go to Exchange Rates screen
2. Click Edit on currency pair
3. Enter new rate
4. Set effective date
5. Add reason (optional)
6. Click Save
7. Previous rate auto-closed
8. New rate becomes active

### Flow 3: Configure Policy
1. Go to Currency Policy
2. Toggle multi-currency settings
3. Configure rate tolerance
4. Set rounding preferences
5. Configure display options
6. Click Save
7. Policy applies to all new expenses

### Flow 4: Bulk Rate Update
1. Click "Bulk Update Rates"
2. Set effective date
3. Enter new rates for all currencies
4. Review change percentages
5. Click Update
6. All rates updated atomically

## Notes for UI Developer

1. **Currency Symbols**
   - Use Unicode characters for currency symbols
   - Ensure proper font support for all symbols
   - Test with RTL currencies (AED, SAR)

2. **Number Formatting**
   - Use Intl.NumberFormat for formatting
   - Respect decimal_places setting
   - Respect separator settings

3. **Rate Precision**
   - Always show 8 decimal places for rates
   - Use DECIMAL input type
   - Avoid floating point issues

4. **Date Handling**
   - Use ISO format for API calls
   - Display in user's locale
   - Handle timezone properly

5. **Accessibility**
   - Add ARIA labels for currency symbols
   - Screen reader support for rate changes
   - Keyboard navigation in bulk update

6. **Performance**
   - Cache currency list
   - Debounce conversion calculations
   - Lazy load rate history
